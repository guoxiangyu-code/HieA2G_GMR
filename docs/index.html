Retrieving Any Relevant Moments

Benchmark and Models for Generalized Moment Retrieval.

We formulate Generalized Moment Retrieval (GMR), a unified video-language retrieval setting where each query may correspond to no moment, one moment, or multiple moments in a video.

GMR requires a system to retrieve the complete set of relevant temporal moments or predict an empty set when the queried event is absent.

Task

Generalized Moment Retrieval

Conventional video moment retrieval usually assumes that every query has exactly one matching segment. Real video search is broader: an event can be absent, occur once, or occur repeatedly in the same video.

01

Null-set Rejection

The query is meaningful, but no corresponding event occurs in the video. The correct output is an empty set.

02

Single-Moment Retrieval

The query corresponds to exactly one temporal segment, matching the classic VMR setting.

03

Multi-Moment Retrieval

The query occurs multiple times, and all relevant moments should be retrieved rather than only the strongest hit.

Pipeline

Duration-Flexible Semi-Automated Data Construction

Soccer-GMR converts timestamped soccer event supervision into generalized moment retrieval annotations through query construction, positive and in-domain negative sampling, temporal normalization, and human verification.

Scaling

From Fixed Clips to Longer Horizons

The construction design supports scaling from 150-second clips to longer video horizons by merging adjacent clips while preserving moment-level supervision.

Metrics

A Unified Evaluation Protocol for GMR

GMR is evaluated through rejection quality, temporal localization on positive queries, and end-to-end generalized retrieval performance.

Null-set Rejection

  • AUROC
  • Rej-F1
  • Accuracy under thresholded existence prediction

Temporal Localization

  • mAP
  • mR@k
  • mR+@k
  • mIoU@k / mIoU+@k

End-to-End GMR

  • G-mIoU@k
  • Joint rejection and localization assessment
  • Unified generalized retrieval view
Evaluation lens Rejection / Localization / End-to-End GMR
Toolkit Official JSONL evaluation scripts are released in the GitHub repository
Null-set Rejection

Rej-F1

Treats correctly rejected null-set queries as the target class at an operating threshold tau.

Rej-F1 = 2TPr / (2TPr + FPr + FNr)
Multi-moment Retrieval

mR+@k

Measures the retrieval of additional relevant moments beyond the first matched moment on multi-moment queries.

mR+@k = 1/|I| sumtheta in I 1/|Qm| sumq in Qm max(0, |Mk(q;theta)| - 1) / (|G(q)| - 1)
End-to-End GMR

G-mIoU@k

Jointly evaluates rejection and localization over all queries after gating predictions by the existence score.

G-mIoU@k = 1/|Q| sumq in Q IoUG(q)
IoUG(q) = 1 for correct empty-set rejection; 0 for positive/null-set mismatch; otherwise set-level IoU between top-k predictions and all ground-truth moments.

Methods

Models for Generalized Moment Retrieval

The paper studies GMR across two complementary modeling paradigms: discriminative VMR backbones with a lightweight adapter and generative MLLMs with a GMR-tailored GRPO reward.

GMR Adapter

A plug-and-play module for discriminative VMR models that adds explicit existence estimation for null-set prediction while keeping the temporal grounding backbone intact.

  • Compatible with mainstream VMR architectures
  • Explicit null-set awareness
  • Improves end-to-end GMR performance

GMR-tailored GRPO Reward

A reward design for GRPO-based fine-tuning of generative MLLMs that jointly encourages temporal localization quality and correct rejection behavior.

  • Targets generalized retrieval objectives
  • Rewards both localization and abstention
  • Extends GMR to generative video-language models
